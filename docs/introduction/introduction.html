GitOps Workflow
STEP 01

代碼與版本管控

所有代碼異動皆透過 GitHub 存儲庫管理。推送到 main 分支後將觸發自動化部屬管線。

• 分支策略：Git Flow / GitHub Flow
• 自動化觸發：GitHub Webhooks

STEP 02

雲端自動化編譯

GCP Cloud Build 接手後，會根據 cloudbuild.yaml 啟動容器化環境，執行 Vite 建置與測試。

• 編譯：Vite 7.1 + React 19
• 環境：Node 24 (Alpine Linux)

STEP 03

鏡像版本管理

建置完成的 Docker 鏡像會被推送到 Artifact Registry，做為穩定的生產環境交付資產。

• 安全掃描：自動弱點偵測
• 版本管理：SHA 標記與快照

STEP 04

無伺服器部署

鏡像自動部署至 Google Cloud Run。支援自動擴展與台北 (asia-east1) 的超低延遲反應。

• 機房：asia-east1 (Taiwan)
• 特性：Serverless, Auto-scaling

STEP 05

全球分發與 PWA

Firebase Hosting 做為前端入口，提供全球 CDN 加速與 HTTPS 加密，並處理 PWA 的靜態資產快取。

• 域名：tw-student.web.app
• 加速：Global Goolge CDN
