# 研究紀錄：本地資料庫預抓與可觀測縣市 Atlas

## 決策一：採用瀏覽器 IndexedDB，而非額外後端

- **結論**：使用瀏覽器原生 IndexedDB 存放已載入的縣市細節與鄉鎮切片。
- **原因**：資料仍是靜態產物，沒有必要為了快取再引入 API 服務。

## 決策二：預抓只限高意圖情境

- **結論**：只在地圖 hover 或排行前幾名時預抓。
- **原因**：能兼顧首頁輕量與體感速度，不會退化成全量背景下載。

## 決策三：來源觀測直接放在摘要與地圖舞台

- **結論**：讓使用者直接看到已載入切片、快取命中與檔案大小。
- **原因**：這次優化不只要更快，也要讓載入策略可理解、可驗證。
