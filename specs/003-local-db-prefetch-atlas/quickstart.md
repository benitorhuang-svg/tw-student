# 快速驗證：SQLite 資料庫驅動的可觀測縣市 Atlas

## 本機執行

1. 在專案根目錄執行 `npm run data:refresh`。
2. 在專案根目錄執行 `npm run dev`。
3. 另開終端執行 `npm run test:e2e`。

## 驗證情境

### 情境一：首頁以 SQLite 啟動

1. 開啟首頁。
2. 檢查 Network。
3. 確認教育資料主要來自單一 `education-atlas.sqlite`，而不是逐縣 `counties/*.json`。

### 情境二：hover 預抓

1. 將滑鼠移到某縣市上。
2. 檢查 Network 與來源觀測區。
3. 確認只有該縣市相關切片被補載。

### 情境三：SQLite 查詢命中

1. 點進某縣市後再回到全台。
2. 再次進入同一縣市。
3. 確認來源觀測區中的 SQLite 命中增加。

### 情境四：學校表格排序與匯出

1. 進入任一縣市。
2. 在學校表格點擊 `學生數`、`年增減`、`狀態` 欄位排序。
3. 點擊 `匯出 CSV` 並確認檔案可下載。

### 情境五：大型縣市完整資料

1. 選擇 `臺中市` 並切到 `學校工作台`。
2. 確認地圖與表格不只顯示大專院校，而能看到國小、國中與高中職資料。
3. 檢查來源觀測區，確認查詢來源為 SQLite。
