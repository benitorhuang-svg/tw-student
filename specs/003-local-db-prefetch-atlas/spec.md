# 功能規格書：本地資料庫預抓與可觀測縣市 Atlas

**功能分支**: `003-local-db-prefetch-atlas`  
**建立日期**: 2026-03-07  
**狀態**: 實作中  
**輸入來源**: 使用者需求「新增 specs 003、建立本地資料庫、只在 hover 或排行前幾名時預抓、學校改成可排序表格並支援匯出、地圖與摘要加入載入來源觀測指標、版型參考 local_view.html」

## 使用者故事與測試情境

### 使用者故事一：只在高意圖情境預抓縣市細節（優先級 P1）

作為分析使用者，我希望系統只在我 hover 某縣市，或該縣市已經進入排行前幾名時才預先載入細節，讓畫面維持輕量，但我真的要點進去時又能更快回應。

**獨立驗證方式**：首頁進站後確認不會載入全部縣市細節；滑過某縣市或觀察排行前幾名後，再查看 Network 與介面觀測區，確認只補載對應縣市切片。

**驗收情境**：

1. **當** 使用者僅停留在全台首頁時，**則** 系統不得同步下載所有縣市細節。
2. **當** 使用者 hover 某個縣市，**則** 系統可在背景預抓該縣市細節與必要鄉鎮切片。
3. **當** 某縣市進入當前排行前幾名時，**則** 系統可在背景預抓其細節，但仍不得擴大成全量下載。

### 使用者故事二：以本地資料庫重用已載入細節（優先級 P1）

作為重複切換縣市的使用者，我希望已載入過的縣市細節能存在瀏覽器本地資料庫，下次切回時直接命中本地快取，減少再次等待。

**獨立驗證方式**：進入某縣市、回到全台、再次點進同一縣市，確認觀測區顯示快取命中，且切換速度明顯快於首次載入。

**驗收情境**：

1. **當** 某縣市細節首次從網路載入完成後，**則** 系統必須將該縣市細節與必要切片寫入本地資料庫。
2. **當** 使用者再次要求相同縣市資源時，**則** 系統應優先嘗試記憶體快取與本地資料庫，而非直接重抓網路。
3. **當** 瀏覽器不支援或拒絕本地資料庫時，**則** 系統仍須保留原本的網路載入能力，不得白畫面。

### 使用者故事三：學校清單升級為可排序表格（優先級 P1）

作為研究者，我希望縣市與鄉鎮的學校清單改成真正的資料表格，讓我能按學生數、年增減、狀態與其他欄位排序，並匯出目前的分析結果。

**獨立驗證方式**：點進任一縣市後確認學校區以表格呈現，點擊欄位標頭可排序，並能匯出 CSV。

**驗收情境**：

1. **當** 使用者進入縣市或鄉鎮模式後，**則** 學校區必須顯示可排序表格，而不是固定順序清單。
2. **當** 使用者切換排序欄位時，**則** 表格順序必須即時更新。
3. **當** 使用者點擊匯出時，**則** 系統必須匯出目前篩選結果的 CSV。

### 使用者故事四：把載入來源與切片大小視覺化（優先級 P2）

作為進階使用者，我希望能直接看到目前已載入哪些縣市切片、快取命中次數與檔案大小，讓我理解系統正在使用哪種載入來源。

**獨立驗證方式**：開啟首頁與切換縣市後，確認地圖與摘要區都能顯示來源觀測資訊。

**驗收情境**：

1. **當** 系統載入縣市細節或鄉鎮切片後，**則** 地圖或摘要區必須更新已載入縣市清單。
2. **當** 載入來自記憶體快取或本地資料庫時，**則** 觀測區必須增加快取命中計數。
3. **當** 顯示已載入切片時，**則** UI 必須能看到對應檔案大小。

## 功能需求

- **FR-001**：系統必須在不破壞摘要先載的前提下加入瀏覽器本地資料庫快取層。
- **FR-002**：預抓策略只能發生在 hover 某縣市或該縣市屬於當前排行前幾名時。
- **FR-003**：系統必須避免同一縣市切片被重複併發下載。
- **FR-004**：學校清單必須改為可排序表格，至少支援學生數、年增減、狀態、鄉鎮、學制、公私立排序。
- **FR-005**：學校表格必須提供 CSV 匯出能力。
- **FR-006**：地圖與摘要區必須顯示已載入縣市切片、快取命中次數、累積傳輸量與檔案大小。
- **FR-007**：資料刷新腳本必須在摘要檔中提供各縣市細節與鄉鎮切片的檔案大小 metadata。
- **FR-008**：版型必須延續 `local_view.html` 的控制台方向，強化地圖舞台、圖例與來源觀測區。

## 成功指標

- **SC-001**：首頁 Network 不會同時下載全部縣市細節檔。
- **SC-002**：重複進入同一縣市時，可在觀測區看到快取命中增加。
- **SC-003**：學校區能完成排序與 CSV 匯出。
- **SC-004**：地圖與摘要面板都能顯示載入來源觀測資訊。
