# 研究紀錄：SQLite 資料庫驅動的可觀測縣市 Atlas

## 決策一：採用 SQLite 檔案作為教育資料主來源，而非逐縣 JSON

- **結論**：用 `sql.js` 在建置階段產出 SQLite，前端直接載入同一份資料庫檔查詢。
- **原因**：可維持靜態部署，同時提供結構化查詢能力，避免多個 JSON 切片分散維護與同步錯誤。

## 決策二：預抓只限高意圖情境

- **結論**：只在地圖 hover 或排行前幾名時預抓。
- **原因**：能兼顧首頁輕量與體感速度，不會退化成全量背景下載。

## 決策三：來源觀測直接放在摘要與地圖舞台

- **結論**：讓使用者直接看到已載入切片、快取命中與檔案大小。
- **原因**：這次優化不只要更快，也要讓載入策略可理解、可驗證。

## 決策四：原始碼必須持續拆成原子模組

- **結論**：把 `App.tsx`、資料層、分析邏輯與刷新腳本切成多個小模組，主檔只保留組裝責任。
- **原因**：SQLite 查詢、邊界載入、觀測記錄、比較情境與分析計算屬於不同責任，集中在單檔會持續製造回歸。
