# 快速驗證：台灣學生人數互動探索平台

## 本機執行

1. 在專案根目錄執行 `npm run data:refresh` 以更新正式資料產物。
2. 在專案根目錄執行 `npm run dev`。
3. 在瀏覽器開啟 Vite 顯示的本機網址。

## 驗證情境

### 情境一：全台 → 縣市 → 鄉鎮下鑽

1. 開啟儀表板。
2. 確認全台摘要、圖例與地圖皆可見。
3. 點擊任一縣市地圖區塊。
4. 確認目前範圍、鄉鎮排行與學校清單同步切換。
5. 再點擊任一鄉鎮，確認學校細節與趨勢圖更新。

### 情境二：URL 狀態同步

1. 切換學年度。
2. 變更教育階段。
3. 選定一個縣市與一個鄉鎮。
4. 確認網址列同步更新。
5. 重新整理頁面，確認狀態可還原。
6. 將網址貼到新分頁，確認顯示相同分析內容。

### 情境三：資料切片與延遲載入

1. 啟動頁面後先停留在全台模式。
2. 透過瀏覽器 Network 面板確認不是一開始就載入所有深層學校細節檔。
3. 點擊縣市後再確認縣市細節切片被載入。

### 情境四：正式資料註記與異常提示

1. 找到具有缺年或異常註記的學校或行政區。
2. 確認細節區可看到缺漏年度、停辦或改制等提示。
3. 確認趨勢圖或資訊卡不會無提示地掩蓋異常。

### 情境五：行動版操作

1. 開啟瀏覽器 devtools 的響應式模式。
2. 驗證控制區、地圖、排行、學校清單與註記區在小尺寸下仍可操作。

## 發版前檢查

1. 在專案根目錄執行 `npm run lint`。
2. 在專案根目錄執行 `npm run build`。
3. 確認 `frontend/public/data/` 下產物皆為多行格式 JSON / TopoJSON。
