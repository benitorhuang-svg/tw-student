# 功能規格書：台灣學生人數互動探索平台

**功能分支**: `001-interactive-edu-map`  
**建立日期**: 2026-03-07  
**狀態**: 已進入優化迭代  
**輸入來源**: 使用者需求「建立高互動性的台灣學生人數分析平台，支援全台 → 縣市 → 鄉鎮市區 → 學校歷年趨勢下鑽，並採用教育部與正式行政區資料」

## 使用者故事與測試情境

### 使用者故事一：真實地理輪廓地圖下鑽分析（優先級 P1）

作為分析人員，我希望能在真實台灣行政區輪廓圖上直接點擊縣市並深入到鄉鎮市區，讓我不用離開同一頁面就能從全台總覽走到地方教育細節。

**優先原因**：這是整個產品的核心價值，也是使用者明確指定的主流程。

**獨立驗證方式**：開啟儀表板後點擊任一縣市，確認頁面在不跳頁的前提下切換到該縣市，再點擊地圖內的鄉鎮，確認排行、摘要、學校清單與趨勢同步更新。

**驗收情境**：

1. **當** 儀表板以全台模式開啟時，**若** 使用者點擊縣市圖形，**則** 介面必須切換為該縣市分析範圍，並顯示對應的鄉鎮與學校模組。
2. **當** 使用者已進入縣市模式時，**若** 點擊鄉鎮圖形或鄉鎮排行，**則** 儀表板必須顯示該鄉鎮的摘要與學校細節。
3. **當** 使用者執行回到全台操作時，**則** 目前篩選條件必須保留，但分析範圍應回到全台模式。

---

### 使用者故事二：可分享的 URL 狀態同步（優先級 P1）

作為決策者，我希望每一次分析條件都能同步到網址列，讓我可以直接分享目前的縣市、鄉鎮、學年度與學制篩選結果給其他人。

**優先原因**：沒有可分享狀態，分析結果就無法被討論、留存或重現。

**獨立驗證方式**：切換學年度、學制、縣市與鄉鎮後重新整理頁面，確認狀態可還原；複製網址到新分頁開啟，確認分析上下文一致。

**驗收情境**：

1. **當** 使用者改變學年度、學制、縣市或鄉鎮時，**則** URL 查詢參數必須同步更新。
2. **當** 使用者以帶參數網址重新進站時，**則** 頁面必須還原相同分析範圍與篩選條件。
3. **當** URL 參數無效、缺漏或指向不存在的範圍時，**則** 系統必須安全回退到最近的有效範圍，不得造成白畫面或錯誤狀態。

---

### 使用者故事三：大資料量載入優化（優先級 P2）

作為一般使用者，我希望正式資料載入速度夠快，讓我第一次開啟頁面時不必等待過久，也不會因完整 GeoJSON 與全量學校資料過大而卡頓。

**優先原因**：正式資料導入後，若沒有切片與壓縮策略，互動體驗會明顯退化。

**獨立驗證方式**：執行資料刷新與正式建置後，確認邊界資料已改用壓縮格式或預切片產物，且頁面仍可正常顯示地圖與細節內容。

**驗收情境**：

1. **當** 建立正式資料產物時，**則** 系統必須輸出適合前端快速載入的壓縮地理資料格式與分段資料檔案。
2. **當** 使用者只停留在全台或縣市層級時，**則** 不應強制一次載入所有深層細節資料。
3. **當** 前端讀取資料檔時，**則** 所有 JSON 產物必須採用多行格式，便於檢視與版本控管。

---

### 使用者故事四：正式資料註記與異常提示（優先級 P2）

作為研究人員，我希望知道哪些學校可能停辦、哪些年度缺漏、哪些行政區名稱或邊界曾改制，避免把資料異常誤解成真實趨勢。

**優先原因**：正式資料的可信度來自透明註記，而不是假設資料永遠完整。

**獨立驗證方式**：切換到存在缺漏或異常標記的學校與行政區時，確認介面能顯示資料註記、缺年資訊或異常說明。

**驗收情境**：

1. **當** 某學校缺少部分學年度資料時，**則** 學校細節區必須顯示缺漏年度提示。
2. **當** 某學校已停辦、改名、整併或無有效學生數時，**則** 系統必須提供可辨識的狀態標記。
3. **當** 行政區存在改制或名稱對照差異時，**則** 資料層與呈現層都必須保留對照資訊與註記能力。

## 邊界情境

- 篩選條件導致某縣市或鄉鎮完全無符合學校資料時，必須呈現空狀態而非殘留舊資料。
- 深連結帶入的縣市或鄉鎮參數若不存在，必須回退至最近可用的上層範圍。
- 某學校只有部分年度數據時，趨勢圖必須可顯示缺漏而不是默默補零且不註記。
- 地理資料若載入失敗，頁面必須提供正式資料錯誤訊息與恢復方式。

## 功能需求

### 功能性需求

- **FR-001**：系統必須以全台灣地圖總覽作為主要入口。
- **FR-002**：系統必須支援使用者在同一頁面中由縣市下鑽至鄉鎮市區與學校層級。
- **FR-003**：系統必須使用正式縣市與鄉鎮市區輪廓資料，而非示意節點圖。
- **FR-004**：系統必須顯示目前分析範圍的學生總數、學校總數、年變化與範圍標籤。
- **FR-005**：系統必須支援學年度、教育階段、公私立、區域群組與關鍵字篩選。
- **FR-006**：系統必須讓地圖、摘要卡、排行、趨勢與學校清單共用同一份篩選狀態。
- **FR-007**：系統必須將學年度、教育階段、縣市、鄉鎮與關鍵篩選條件同步到 URL。
- **FR-008**：系統必須能從 URL 還原分享連結的分析狀態。
- **FR-009**：系統必須將正式資料產製為適合前端載入的切片與壓縮格式，避免首次載入過重。
- **FR-010**：系統必須保留學校層級細節，包括校名、區域、學制、公私立、當年度學生數與歷年趨勢。
- **FR-011**：系統必須支援正式資料註記，包括停辦、缺年度、名稱異動、行政區改制差異等訊息。
- **FR-012**：系統必須在桌面與行動裝置上維持可用的完整互動流程。
- **FR-013**：系統必須輸出多行格式的 `.json` 產物，不得寫成單行壓縮內容。
- **FR-014**：系統必須維持前端資料契約，讓資料刷新腳本、地理資料產物與 UI 模組可獨立替換而不破壞互動流程。

### 關鍵實體

- **縣市**：第一層行政範圍，包含區域群組、邊界幾何、彙總指標與鄉鎮集合。
- **鄉鎮市區**：縣市內的第二層行政範圍，包含邊界幾何、彙總指標與學校集合。
- **學校**：可呈現的教育單位，包含學制、公私立、座標、聯絡資訊、歷年學生數與資料狀態註記。
- **篩選狀態**：包含學年度、學制、公私立、區域群組、關鍵字、選定縣市與選定鄉鎮。
- **資料註記**：描述停辦、缺年、異常值、名稱異動或行政區對照差異的結構。
- **分享狀態**：可序列化到 URL 的分析上下文。

## 成功指標

### 可衡量成果

- **SC-001**：使用者可在一次點擊內從全台進入縣市細節，並在第二次點擊內進入鄉鎮層級。
- **SC-002**：使用者複製分析網址到新分頁後，可還原相同縣市、鄉鎮、學年度與學制狀態。
- **SC-003**：正式地理資料與學校資料優化後，正式建置仍可成功完成，且互動畫面不因資料量提升而失效。
- **SC-004**：使用者能清楚辨識學校停辦、缺漏年度或行政區改制等資料異常訊息。
- **SC-005**：所有規格文件與驗證說明皆維持繁體中文。
