# 規格：預產生 bucket 校點、可搬移比較情境與異常批次治理

## 目標

- 保持開發模式穩定，避免 `react-leaflet` 與 Vite 預打包再次導致瀏覽器載入失敗。
- 將縣市模式下的學校點位分群結果下放到資料刷新腳本，直接輸出縣市級 geohash bucket JSON，前端只負責 lazy 載入與視窗過濾。
- 將比較工作台升級為可分享、可收藏、可重新命名、可釘選，並能匯出 / 匯入 JSON 的比較情境庫。
- 在異常資料調查面板中提供單筆原始年度序列檢視、單筆下載、整批匯出與匯出篩選，讓缺年度、停辦與待確認可直接對照原始切片。
- 依據探索數據文件，強化 Macro / Regional / Micro 三層資訊架構，使頁面更清楚區分全台總覽、區域下鑽與學校工作台。
- 將頁籤移入左欄篩選區上方，形成穩定的「左欄導覽與篩選 / 右欄主畫面」工作台版型。
- `回到全台` 必須重新定位到台灣本體視角，不得落到外海或境外底圖範圍。
- 區域分析頁的地圖互動應專注在縣市 → 鄉鎮下鑽，不得混入學校校點分群造成視覺干擾。
- 全台與縣市探索時，學校點位應以圓點呈現，並依教育階段提供穩定色碼，不得讓低縮放畫面被大量 `1` 分群標記切碎。
- 資料來源列必須可點擊前往官方網站，並提供手動資料更新按鈕，讓使用者能主動重抓最新靜態資料。
- 右側主畫面需移除常駐的冗餘快取 / 來源觀測卡片，改用地圖右上角燈泡提示整合探索說明與圖例。

## 使用者故事

1. 作為分析人員，我希望切進單一縣市後能直接使用資料刷新腳本產出的 bucket 切片，不必等待前端重新計算，拖曳或縮放時能更穩定探索高密度學校區塊。
2. 作為研究協作者，我希望比較情境除了寫進網址，也能重新命名、釘選重要情境，並以 JSON 匯出 / 匯入到另一台裝置。
3. 作為資料治理使用者，我希望整批匯出異常序列時能先挑選 `缺年度`、`待確認` 或其他類型，減少人工整理時間。
4. 作為決策者，我希望頁面能清楚區分全台總覽、區域下鑽與學校工作台三種層級，快速理解目前所處的分析視角。
5. 作為維護者，我希望在開發模式啟動時不會再遇到 `react-leaflet` 預打包錯誤，也不會因 frontend 套件未安裝而導致啟動失敗。

## 驗收條件

- 根目錄 `npm install` 後會自動安裝 frontend 依賴，`npm run dev` 不再因 React / Vite 預打包或 `leaflet/dist/leaflet.css` 問題而失敗。
- `frontend/scripts/refresh-official-data.mjs` 會為每個縣市輸出獨立的 geohash bucket JSON，摘要資料同步帶出 bucket 檔案路徑與大小資訊。
- 切進縣市後，地圖低縮放層級讀取 bucket 切片，高縮放層級顯示單校點位；前端不再於載入後重算全部 bucket。
- 比較工作台支援 `compare` 與 `scenario` query 參數，並提供本地收藏、重新命名、釘選、最近開啟，以及 JSON 匯出 / 匯入。
- 異常調查面板可選取一筆異常，查看原始年度序列、下載單筆 CSV，並按類型篩選後整批匯出。
- UI 會以 Macro / Regional / Micro 的資訊層級重組導覽與摘要，讓目前分析視角更清晰。
- 左欄最上方會固定顯示三個頁籤，頁籤位置必須在學年度與其他篩選器之上。
- 概況總覽頁以全台 / 縣市層級摘要為主；區域分析頁以縣市 → 鄉鎮下鑽與異常治理為主；學校工作台才顯示校點與單校明細。
- 觸發 `回到全台` 後，Leaflet 視角需回到台灣本體 bounds，且頁面狀態同步重設為概況總覽。
- 地圖右上角在 `+ / -` 控制下方必須提供燈泡按鈕；點擊後顯示探索提示與依當前模式切換的圖例說明。
- Footer 的 `資料來源` 必須拆成可點擊的官方連結，並在左側提供 `資料更新` 按鈕。
- 右側地圖區不保留常駐快取摘要、載入來源觀測與圖資來源卡片；冗餘資訊需從主舞台移除。
- `lint`、`build`、`test:e2e` 全數通過。
