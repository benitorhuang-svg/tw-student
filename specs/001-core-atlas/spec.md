# 核心體驗規格：全台學生數地圖分析工作台

**規格編號**: `001-core-atlas`  
**更新日期**: 2026-03-08  
**狀態**: 現行主規格  
**彙整來源**: 原 `001`、`002`、`005`、`006` 系列規格

## 規格定位

本文件定義目前產品的核心互動體驗，將原本依時間演進堆疊的多份規格，整理為一份可持續維護的主線文件。重點是把地圖重新定位成「左側地理篩選器」，右側專注呈現摘要卡片、圖表、排行、學校表格與異常治理。

## 使用者故事與測試情境

### 使用者故事一：左欄地理篩選、右欄分析畫布（優先級 P1）

作為分析人員，我希望左側固定放置頁面導覽、篩選器與地圖，右側集中呈現卡片、圖表與表格，讓我先縮小範圍，再閱讀分析內容，而不是在同一區塊同時處理地圖與資訊。

**優先原因**：這是目前最直接的 UX 痛點，若資訊架構錯置，再多功能也難以使用。

**獨立驗證方式**：開啟頁面後，確認主標題位於頁面內容上方而非獨立 head bar；左側可直接完成頁籤切換、年份與範圍篩選、地圖點選；右側會依目前頁籤切換對應分析內容。

**驗收情境**：

1. **當** 首頁載入完成時，**則** 介面不得再出現獨立的頂部 head bar，主標題必須置於頁面主體上方。
2. **當** 使用者需要縮小分析範圍時，**則** 左側必須同時提供頁籤、條件篩選與行政區地圖。
3. **當** 使用者切換不同分析頁籤時，**則** 右側主畫面必須切換為該頁籤對應的卡片、圖表與表格內容。

---

### 使用者故事二：地圖下鑽與可分享狀態（優先級 P1）

作為決策者，我希望能在同一頁面中由全台 → 縣市 → 鄉鎮下鑽，並把目前條件同步到 URL，讓我可以直接分享分析中的具體範圍。

**優先原因**：下鑽與分享仍是產品核心，不因版型改版而退化。

**獨立驗證方式**：點擊縣市、鄉鎮、切換年份與教育階段後重新整理頁面，確認畫面可由 URL 還原。

**驗收情境**：

1. **當** 儀表板以全台模式開啟時，**若** 使用者點擊縣市圖形，**則** 系統必須切換為該縣市分析範圍。
2. **當** 使用者已進入縣市模式時，**若** 點擊鄉鎮圖形或鄉鎮排行，**則** 右側分析內容必須同步顯示該鄉鎮摘要與學校細節。
3. **當** 使用者改變學年度、教育階段、縣市或鄉鎮時，**則** URL 查詢參數必須同步更新，且重新整理後可還原。

---

### 使用者故事三：學年度以西元顯示（優先級 P1）

作為一般使用者，我希望畫面中的學年度以西元為主，避免看到 `113` 這類需自行轉換的年度格式，提升理解速度。

**優先原因**：年份是最高頻的操作條件之一，若顯示不直觀，整個分析流程會持續增加認知負擔。

**獨立驗證方式**：檢查年份下拉、摘要卡、趨勢圖與異常明細中的年度文案，確認以西元學年度呈現。

**驗收情境**：

1. **當** 頁面顯示當前學年度時，**則** 主顯示必須以西元學年度為主。
2. **當** 趨勢圖或異常表格顯示歷年資料時，**則** 年度標籤不得只顯示民國年數值。
3. **當** URL 仍以既有學年度數值儲存時，**則** 系統仍須能正確解析並轉成西元顯示。

---

### 使用者故事四：分析與治理資訊集中於右側（優先級 P2）

作為研究人員，我希望摘要卡片、長條圖、趨勢圖、學年對照表與異常治理面板都集中在右側，形成穩定的分析工作區。

**優先原因**：這些內容屬於閱讀與比對，不應與篩選器爭奪同一欄位。

**獨立驗證方式**：切換三個頁籤，確認右側依序呈現概況摘要、區域比較 / 異常治理、學校工作台等內容。

**驗收情境**：

1. **當** 目前頁籤為概況總覽時，**則** 右側需呈現範圍摘要、教育分布、趨勢與排行資訊。
2. **當** 目前頁籤為區域分析時，**則** 右側需呈現縣市比較與異常資料治理內容。
3. **當** 目前頁籤為學校工作台時，**則** 右側需呈現學校表格、單校焦點與正式註記。

## 邊界情境

- 篩選條件導致某縣市或鄉鎮完全無符合學校資料時，必須呈現空狀態而非殘留舊資料。
- 深連結帶入的縣市或鄉鎮參數若不存在，必須回退至最近可用的上層範圍。
- 某學校只有部分年度數據時，趨勢圖必須可顯示缺漏而不是默默補零且不註記。
- 地理資料若載入失敗，頁面必須提供正式資料錯誤訊息與恢復方式。
- 行動裝置版面必須能改為單欄堆疊，但仍保留「先篩選、後閱讀」的順序。

## 功能需求

### 功能性需求

- **FR-001**：系統必須以全台灣地圖總覽作為主要入口，並支援縣市 → 鄉鎮 → 學校層級下鑽。
- **FR-002**：系統必須將頁籤、篩選器與地圖整合在左側控制欄。
- **FR-003**：系統必須將摘要卡片、比較圖表、趨勢圖、異常治理與學校表格集中在右側分析畫布。
- **FR-004**：系統必須移除獨立 head bar，並將主標題移入頁面內容區上方。
- **FR-005**：系統必須支援學年度、教育階段、公私立、區域群組與關鍵字篩選。
- **FR-006**：系統必須讓地圖、摘要卡、排行、趨勢與學校清單共用同一份篩選狀態。
- **FR-007**：系統必須將學年度、教育階段、縣市、鄉鎮與關鍵篩選條件同步到 URL。
- **FR-008**：系統必須能從 URL 還原分享連結的分析狀態。
- **FR-009**：系統中的學年度主要顯示必須以西元學年度為主，不得只顯示民國年。
- **FR-010**：系統必須保留學校層級細節，包括校名、區域、學制、公私立、當年度學生數與歷年趨勢。
- **FR-011**：系統必須支援正式資料註記，包括停辦、缺年度、名稱異動、行政區改制差異等訊息。
- **FR-012**：系統必須在桌面與行動裝置上維持可用的完整互動流程。
- **FR-013**：系統必須將正式資料產製為適合前端載入的切片與壓縮格式，避免首次載入過重。
- **FR-014**：系統必須輸出多行格式的 `.json` 產物，不得寫成單行壓縮內容。
- **FR-015**：系統必須維持前端資料契約，讓資料刷新腳本、地理資料產物與 UI 模組可獨立替換而不破壞互動流程。

### 關鍵實體

- **縣市**：第一層行政範圍，包含區域群組、邊界幾何、彙總指標與鄉鎮集合。
- **鄉鎮市區**：縣市內的第二層行政範圍，包含邊界幾何、彙總指標與學校集合。
- **學校**：可呈現的教育單位，包含學制、公私立、座標、聯絡資訊、歷年學生數與資料狀態註記。
- **篩選狀態**：包含學年度、學制、公私立、區域群組、關鍵字、選定縣市與選定鄉鎮。
- **控制欄**：左側固定區域，包含頁籤、篩選器與地圖，負責縮小分析範圍。
- **分析畫布**：右側內容區域，依頁籤切換摘要、比較、趨勢、表格與治理面板。
- **資料註記**：描述停辦、缺年、異常值、名稱異動或行政區對照差異的結構。
- **分享狀態**：可序列化到 URL 的分析上下文。

## 成功指標

### 可衡量成果

- **SC-001**：使用者可在一次點擊內從全台進入縣市細節，並在第二次點擊內進入鄉鎮層級。
- **SC-002**：使用者複製分析網址到新分頁後，可還原相同縣市、鄉鎮、學年度與學制狀態。
- **SC-003**：首次接觸頁面時，使用者可直接理解左側為篩選與地理縮放、右側為分析內容。
- **SC-004**：所有主要年份顯示皆以西元學年度呈現，降低誤讀與心智轉換成本。
- **SC-005**：使用者能清楚辨識學校停辦、缺漏年度或行政區改制等資料異常訊息。
- **SC-006**：所有規格文件與驗證說明皆維持繁體中文。
