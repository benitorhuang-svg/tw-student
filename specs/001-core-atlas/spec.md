# 核心體驗規格：全台學生數地圖分析工作台

**規格編號**: `001-core-atlas`  
**更新日期**: 2026-03-09  
**狀態**: 現行主規格  
**彙整來源**: 原 `001`、`002`、`005`、`006` 系列規格

## 規格定位

本文件定義目前產品的核心互動體驗，將原本依時間演進堆疊的多份規格，整理為一份可持續維護的主線文件。重點已改為 Power BI 風格的一頁式儀表板：地圖與篩選器負責驅動單一分析畫布，主要聚焦就讀人數、年度變化、教育階段分布與行政區排行，不再依賴多頁籤切換或離線 / 快取狀態面板。

## 使用者故事與測試情境

### 使用者故事一：單頁儀表板與篩選驅動互動（優先級 P1）

作為分析人員，我希望畫面以單頁儀表板呈現，透過上方或抽屜式篩選器搭配地圖互動來更新同一張分析畫布，避免靠頁籤切換或長頁面捲動才能完成判讀。

**優先原因**：這是目前最直接的 UX 痛點，若資訊架構錯置，再多功能也難以使用。

**獨立驗證方式**：開啟頁面後，確認儀表板維持單頁、無頁面主捲動；篩選器、地圖點選與排行點擊都會直接更新同一套圖表與摘要卡。

**驗收情境**：

1. **當** 首頁載入完成時，**則** 介面必須直接顯示單頁儀表板，不得要求使用者先切換頁籤才看到核心分析。
2. **當** 使用者需要縮小分析範圍時，**則** 必須可透過篩選器、地圖點擊與排行列點選完成互動。
3. **當** 使用者改變學年度、教育階段、公私立、區域或關鍵字時，**則** 儀表板中的摘要卡、圓餅圖、趨勢圖與排行必須同步更新。

---

### 使用者故事二：地圖下鑽與可分享狀態（優先級 P1）

作為決策者，我希望能在同一頁面中由全台 → 縣市 → 鄉鎮下鑽，並把目前條件同步到 URL，讓我可以直接分享分析中的具體範圍。

**優先原因**：下鑽與分享仍是產品核心，不因版型改版而退化。

**獨立驗證方式**：點擊縣市、鄉鎮、切換年份與教育階段後重新整理頁面，確認畫面可由 URL 還原。

**驗收情境**：

1. **當** 儀表板以全台模式開啟時，**若** 使用者點擊縣市圖形，**則** 系統必須切換為該縣市分析範圍。
2. **當** 使用者已進入縣市模式時，**若** 點擊鄉鎮圖形或鄉鎮排行，**則** 右側分析內容必須同步顯示該鄉鎮摘要與學校細節。
3. **當** 使用者改變學年度、教育階段、縣市或鄉鎮時，**則** URL 查詢參數必須同步更新，且重新整理後可還原。

---

### 使用者故事三：學年度顯示需附民國與西元區間（優先級 P1）

作為一般使用者，我希望畫面中的學年度同時顯示民國學年度與對應西元區間，例如 `114學年度 (2025-2026)`，避免自行換算。

**優先原因**：年份是最高頻的操作條件之一，若顯示不直觀，整個分析流程會持續增加認知負擔。

**獨立驗證方式**：檢查年份下拉與主要年度文案，確認至少在主要篩選列中顯示 `民國學年度 + 西元區間` 的格式。

**驗收情境**：

1. **當** 頁面顯示當前學年度時，**則** 主顯示必須至少包含 `114學年度 (2025-2026)` 這類完整格式。
2. **當** 趨勢圖或異常表格顯示歷年資料時，**則** 至少不得讓主要篩選列與關鍵提示只顯示裸露的民國數字而無輔助說明。
3. **當** URL 仍以既有學年度數值儲存時，**則** 系統仍須能正確解析並轉成完整顯示格式。

---

### 使用者故事四：分析內容聚焦就讀人數（優先級 P2）

作為研究人員，我希望儀表板主要聚焦就讀人數分析，不再被離線、快取或過度治理性資訊干擾，讓趨勢、分布與排行能更快形成判讀。

**優先原因**：這些內容屬於閱讀與比對，不應與篩選器爭奪同一欄位。

**獨立驗證方式**：切換不同篩選條件與行政區層級，確認核心畫布只保留學生數摘要、教育階段分布、年度趨勢與排行。

**驗收情境**：

1. **當** 目前顯示全台模式時，**則** 核心畫布需呈現全台學生總量、教育分布、歷年趨勢與縣市排行。
2. **當** 使用者切入縣市模式時，**則** 核心畫布需同步切成該縣市學生總量、教育分布、歷年趨勢與鄉鎮排行。
3. **當** 使用者切入鄉鎮模式時，**則** 同一張畫布需聚焦該鄉鎮摘要與學生數趨勢，不得另開第二套分頁流程。

## 邊界情境

- 篩選條件導致某縣市或鄉鎮完全無符合學校資料時，必須呈現空狀態而非殘留舊資料。
- 深連結帶入的縣市或鄉鎮參數若不存在，必須回退至最近可用的上層範圍。
- 某學校只有部分年度數據時，趨勢圖必須可顯示缺漏而不是默默補零且不註記。
- 地理資料若載入失敗，頁面必須提供正式資料錯誤訊息與恢復方式。
- 行動裝置版面必須能改為單欄堆疊，但仍保留「先篩選、後閱讀」的順序，且篩選器需可透過抽屜開啟。
- 深連結若帶入 `zoom`、`lat`、`lon`，畫面重整後必須保留相同地圖視窗，且使用者可直接在地圖浮動工具看到目前視窗值。
- 鄉鎮標記在 11 到 12 級縮放時若發生密集重疊，系統必須優先保留學生數較高的標記，避免重要區塊完全不可讀。

## 功能需求

### 功能性需求

- **FR-001**：系統必須以全台灣地圖總覽作為主要入口，並支援縣市 → 鄉鎮 → 學校層級下鑽。
- **FR-002**：系統必須以單頁儀表板呈現核心分析，不再依賴多頁籤切換主要內容。
- **FR-003**：系統必須讓篩選器、地圖、摘要卡、教育分布、趨勢圖與排行共享同一份互動狀態。
- **FR-004**：系統必須避免主要頁面產生多層滾動，桌面版以固定視窗單頁為原則，行動版則以抽屜篩選維持單欄閱讀。
- **FR-005**：系統必須支援學年度、教育階段、公私立、區域群組與關鍵字篩選。
- **FR-006**：系統必須讓地圖、摘要卡、排行與趨勢共用同一份篩選狀態。
- **FR-007**：系統必須將學年度、教育階段、縣市、鄉鎮與關鍵篩選條件同步到 URL。
- **FR-008**：系統必須能從 URL 還原分享連結的分析狀態。
- **FR-009**：系統中的學年度主要顯示必須同時提供民國學年度與對應西元區間，不得只顯示單一裸露年份。
- **FR-010**：系統必須保留行政區與學校層級的正式資料基礎，以支援地圖下鑽與未來延伸，但目前主畫面聚焦在學生數分析。
- **FR-011**：系統不得在主畫面展示離線、快取或資料傳輸量等冗餘狀態面板。
- **FR-012**：系統必須在桌面與行動裝置上維持可用的完整互動流程。
- **FR-013**：系統必須將正式資料產製為適合前端載入的切片與壓縮格式，避免首次載入過重。
- **FR-014**：系統必須輸出多行格式的 `.json` 產物，不得寫成單行壓縮內容。
- **FR-015**：系統必須維持前端資料契約，讓資料刷新腳本、地理資料產物與 UI 模組可獨立替換而不破壞互動流程。
- **FR-016**：系統必須在地圖浮動工具中提供精簡的 viewport debug chip，至少顯示目前 `zoom`、`lat`、`lon`，方便驗證 deep link 是否還原成功。
- **FR-017**：系統必須把缺少座標的學校清單轉為可追蹤的補件工作流，至少支援 `GIS缺點位`、`人工補點`、`已回填` 三種狀態與狀態篩選。
- **FR-018**：系統必須在鄉鎮標記過密時套用碰撞規則或優先級排序，以學生數較高的標記優先保留可見性。
- **FR-019**：當關鍵字搜尋唯一命中正式學校代碼時，系統必須自動同步縣市、鄉鎮與單校選取狀態，讓地圖直接飛到學校座標，並切換右側內容到對應的單校圖表。
- **FR-020**：系統必須採用雙軌狀態模型，將地理範圍 `全台 / 縣市 / 鄉鎮` 與單校聚焦分離管理；區域篩選不得在縣市、鄉鎮或單校操作時被隱性改寫，且縣市層直接選取單校時地圖仍必須顯示對應校點。
- **FR-021**：當關鍵字搜尋唯一命中正式學校代碼時，系統必須至少同步縣市與單校聚焦；若未明確進入鄉鎮範圍，breadcrumb 只呈現地理範圍，不得把單校聚焦誤當成鄉鎮 scope。

### 關鍵實體

- **縣市**：第一層行政範圍，包含區域群組、邊界幾何、彙總指標與鄉鎮集合。
- **鄉鎮市區**：縣市內的第二層行政範圍，包含邊界幾何、彙總指標與學校集合。
- **學校**：可呈現的教育單位，包含學制、公私立、座標、聯絡資訊、歷年學生數與資料狀態註記。
- **篩選狀態**：包含學年度、學制、公私立、區域群組、關鍵字、選定縣市與選定鄉鎮。
- **控制列 / 篩選抽屜**：桌面版位於上方，行動版以抽屜呈現，負責套用篩選條件與快速重設範圍。
- **分析畫布**：單一內容區域，集中顯示學生數摘要、教育分布、趨勢圖與行政區排行。
- **資料註記**：描述停辦、缺年、異常值、名稱異動或行政區對照差異的結構。
- **分享狀態**：可序列化到 URL 的分析上下文。
- **座標補件工作流**：針對缺少 GIS 校點座標的學校所建立的前端追蹤狀態，支援人工補件與回填確認。

## 成功指標

### 可衡量成果

- **SC-001**：使用者可在一次點擊內從全台進入縣市細節，並在第二次點擊內進入鄉鎮層級。
- **SC-002**：使用者複製分析網址到新分頁後，可還原相同縣市、鄉鎮、學年度與學制狀態。
- **SC-003**：首次接觸頁面時，使用者可直接理解左側為篩選與地理縮放、右側為分析內容。
- **SC-004**：所有主要年份顯示皆提供民國學年度與對應西元區間，降低誤讀與心智轉換成本。
- **SC-005**：使用者能清楚辨識學校停辦、缺漏年度或行政區改制等資料異常訊息。
- **SC-006**：所有規格文件與驗證說明皆維持繁體中文。
- **SC-007**：使用者不需開啟 devtools，即可直接從地圖浮動工具讀取目前 `zoom / lat / lon`，確認 deep link 視窗是否正確還原。
- **SC-008**：在高密度縣市縮放至 11 到 12 級時，鄉鎮標記不會因全面重疊而失去可讀性。
